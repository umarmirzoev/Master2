## О проекте

**Мастер Час** — это веб-платформа для поиска и заказа домашних мастеров, а также покупки сопутствующих товаров для ремонта и дома. Проект сочетает в себе:

- маркетплейс услуг;
- каталог мастеров;
- интернет-магазин товаров;
- личные кабинеты для клиента, мастера, администратора и супер-администратора;
- интеграцию с Supabase для авторизации, базы данных и realtime-обновлений;
- AI-подбор мастера под описание проблемы.

Основная идея проекта: пользователь заходит на сайт, выбирает услугу или описывает проблему, система помогает подобрать мастера, оформить заказ, отслеживать статус выполнения и при необходимости купить товары с установкой.

Проект ориентирован на сервис бытовых услуг в Душанбе и использует локализованный интерфейс с поддержкой нескольких языков.

## Что умеет проект

Платформа покрывает несколько сценариев работы:

- заказ мастера по категориям услуг;
- просмотр списка мастеров и их профилей;
- просмотр детальных страниц услуг и категорий;
- оформление заявок на выполнение работ;
- регистрация и вход пользователей;
- подача заявки на получение роли мастера;
- управление корзиной товаров;
- просмотр магазина и карточек товаров;
- оформление заказов с дополнительной опцией установки;
- получение realtime-обновлений по заказам;
- уведомления внутри интерфейса;
- работа с ролями и разными кабинетами;
- AI-анализ проблемы пользователя для подбора мастера.

## Для кого предназначен проект

В системе предусмотрены несколько ролей:

- **Клиент** — ищет мастеров, оформляет заказы, покупает товары, управляет корзиной и следит за статусом заказов.
- **Мастер** — получает доступ к мастер-кабинету, управляет заказами, профилем, товарами и своей активностью.
- **Администратор** — управляет заявками, заказами, мастерами, магазином и внутренними процессами.
- **Супер-администратор** — расширенная административная роль с доступом к отдельной панели управления.

Роли хранятся в Supabase и загружаются в приложении через контекст авторизации.

## Как работает приложение

### 1. Запуск

Приложение стартует через `src/main.tsx`, где React монтируется в DOM-элемент `#root`.

Далее подключается `App.tsx`, который оборачивает всё приложение в провайдеры:

- `QueryClientProvider`;
- `LanguageProvider`;
- `AuthProvider`;
- `CartProvider`;
- `TooltipProvider`.

После этого запускается `BrowserRouter`, а маршруты рендерятся через `Routes` и `Route`.

### 2. Маршрутизация

Маршруты настроены в `src/App.tsx`.

Примеры маршрутов:

- `/` — главная страница;
- `/about` — о проекте (включая "Как это работает");
- `/categories` — категории услуг;
- `/category/:id` — детальная страница категории;
- `/service/:id` — детальная страница услуги;
- `/masters` — список мастеров;
- `/master/:id` и `/masters/:id` — профиль/детали мастера;
- `/shop` — магазин;
- `/shop/category/:id` — категория магазина;
- `/shop/product/:id` — карточка товара;
- `/cart` — корзина;
- `/auth` — вход и регистрация;
- `/dashboard` — кабинет клиента;
- `/master-dashboard` — кабинет мастера;
- `/admin/dashboard` — кабинет администратора;
- `/super-admin/dashboard` — кабинет супер-администратора.

Переходы между страницами анимируются через Framer Motion.

### 3. Авторизация

Авторизация реализована в `src/hooks/useAuth.tsx`.

Что делает этот модуль:

- подписывается на изменение состояния авторизации через `supabase.auth.onAuthStateChange`;
- загружает текущую сессию;
- хранит объект пользователя и сессии;
- подгружает роли из таблицы `user_roles`;
- подгружает профиль из таблицы `profiles`;
- определяет путь, куда отправлять пользователя в зависимости от роли;
- умеет выполнять выход из аккаунта;
- позволяет повторно загрузить данные пользователя.

### 4. Корзина и магазин

Логика корзины находится в `src/hooks/useCart.tsx`.

Что реализовано:

- загрузка содержимого корзины пользователя;
- добавление товара в корзину;
- удаление товара;
- изменение количества;
- включение и выключение опции установки;
- очистка корзины;
- расчёт общего количества товаров;
- расчёт общей стоимости заказа.

Данные корзины связаны с таблицей `cart_items`, а товары подтягиваются из `shop_products`.

### 5. Локализация

Локализация реализована через `LanguageContext.tsx`.

Поддерживаются языки:

- `ru` — русский;
- `tj` — таджикский;
- `en` — английский.

Контекст:

- хранит выбранный язык;
- сохраняет язык в `localStorage` по ключу `mc_lang`;
- предоставляет функцию `t(key)` для получения перевода;
- содержит большой словарь интерфейсных строк.

### 6. Realtime

Хук `useRealtimeOrders.tsx` подписывается на изменения таблицы заказов через канал Supabase Realtime.

Что это даёт:

- клиент видит обновление статуса заказа;
- мастер может получать уведомления о доступных заказах;
- администратор может видеть новые заказы почти мгновенно;
- интерфейс показывает toast-уведомления при изменениях.

### 7. AI-подбор мастера

В проекте есть компонент и серверная функция для AI-подбора мастера:

- `src/components/AiMasterMatch.tsx`;
- `supabase/functions/ai-match-master/index.ts`.

Идея механизма:

- пользователь описывает проблему естественным языком;
- система пытается определить категорию, тип услуги, срочность и релевантных мастеров;
- затем интерфейс предлагает подходящих специалистов.

## Edge Functions Supabase

В проекте присутствуют следующие серверные функции:

- `ai-match-master` — AI-подбор мастера по описанию проблемы;
- `get-products` — API товаров с фильтрами и пагинацией;
- `seed-demo-accounts` — заполнение демо-аккаунтов;
- `seed-demo-data` — заполнение демо-данных.

Согласно `supabase/config.toml`, эти функции работают с отключённой обязательной проверкой JWT на уровне конфигурации.

## Работа с базой данных

Схема базы данных развивается через миграции в папке `supabase/migrations`.

Из кода видно, что приложение использует, как минимум, следующие таблицы и сущности:

- `profiles`;
- `user_roles`;
- `cart_items`;
- `shop_products`;
- `orders`.

Также в типах Supabase присутствует enum ролей `app_role`, который участвует в маршрутизации и разграничении доступа.

## Переменные окружения

Для работы проекта используется файл `.env`.

Основные переменные:

- `VITE_SUPABASE_URL` — URL проекта Supabase;
- `VITE_SUPABASE_PUBLISHABLE_KEY` — публичный ключ Supabase;
- `VITE_SUPABASE_PROJECT_ID` — ID проекта Supabase.

Важно:

- переменные с префиксом `VITE_` доступны на клиенте;
- публиковать приватные сервисные ключи в клиентской части нельзя;
- для production-окружения лучше использовать отдельные значения переменных.

## Что особенно важно знать разработчику

### 1. Это не просто лендинг

Проект уже имеет признаки полноценной продуктовой системы:

- многостраничная архитектура;
- роли и доступы;
- магазин;
- корзина;
- realtime;
- административные панели;
- интеграция с базой данных;
- edge functions.

### 2. Основная бизнес-логика завязана на Supabase

Если что-то не работает, в первую очередь стоит проверять:

- переменные окружения;
- таблицы и миграции Supabase;
- edge functions;
- права доступа и роли;
- наличие данных в `profiles`, `user_roles`, `orders`, `cart_items`, `shop_products`.

### 3. В проекте есть локализация

При добавлении новых экранов и компонентов лучше сразу:

- добавлять новые ключи в словари переводов;
- не хардкодить строки напрямую;
- использовать функцию `t(...)`.

### 4. Есть разделение на уровни кабинетов

Нельзя рассматривать `/dashboard`, `/master-dashboard`, `/admin/dashboard` и `/super-admin/dashboard` как одну и ту же страницу. Это разные зоны с разной логикой и уровнем доступа.

## Краткий итог

**Мастер Час** — это frontend-приложение на React + TypeScript с backend-интеграцией через Supabase. Проект предназначен для сервиса бытовых услуг и магазина товаров для дома. Внутри уже реализованы маршрутизация, роли пользователей, личные кабинеты, локализация, корзина, realtime-обновления и AI-подбор мастера.
